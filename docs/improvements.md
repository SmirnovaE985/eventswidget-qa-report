# Предложения по улучшению

## 1. Улучшение выбора цветовой темы

### Проблема
В блоке выбора темы пользователь видит только цветные кружки, но не видит текстовых названий конкретных вариантов темы:

- синяя
- зелёная
- красная
- бирюзовая
- фиолетовая

### Почему это важно
- пользователь ориентируется только по цвету;
- неочевидно, какое именно системное значение темы будет применено в generated code;
- сложнее составлять точные инструкции, чек-листы и тест-кейсы;
- снижается доступность интерфейса.

### Предложение
Добавить текстовые подписи или tooltip для цветовых тем.

Пример:
- Светлая: Синяя / Зелёная / Красная
- Тёмная: Бирюзовая / Фиолетовая

## 2. Улучшение прозрачности параметров generated code

### Проблема
Для пользователя и тестировщика неочевидно, что:
- `event_type` — это выбранные тематики;
- `theme` — это цветовая тема.

### Почему это важно
- без анализа кода или дополнительной документации можно неверно интерпретировать параметры;
- интерфейс не объясняет, как настройки маппятся в generated code;
- повышается вероятность ошибок при ручной проверке и интеграции.

### Предложение
Добавить пояснение или подсказку рядом с generated code либо в интерфейсе настроек.

Пример:
- `theme` — цветовая гамма виджета
- `event_type` — категории / тематики событий

## 3. Улучшение поведения preview для относительных размеров

### Проблема
Поведение опций **«на всю ширину контейнера»** и **«на всю высоту блока»** неочевидно: generated code использует относительные размеры `100%`, но preview не демонстрирует ожидаемое растяжение виджета.

### Почему это важно
- пользователь может ожидать один результат, а видеть другой;
- возникает рассинхрон между настройками и визуальным предпросмотром;
- это может вводить в заблуждение относительно итогового отображения виджета после вставки на сайт.

### Предложение
- синхронизировать preview с выбранными настройками размеров;
- либо явно указать в интерфейсе, что значения `100%` применяются относительно внешнего контейнера на стороне сайта, а preview отображается в фиксированной демонстрационной области;
- добавить поясняющий tooltip или текст рядом с опциями размеров.

## 4. Улучшение наблюдаемости генерации preview

### Проблема
В DevTools Network при взаимодействии со страницей фиксируются в основном служебные запросы аналитики Яндекс.Метрики (`mc.yandex.ru`), содержащие параметры кликов и браузерной информации (`pointer-click`, `browser-info`, `wv-data`, `wv-part`). По этим запросам нельзя оценить корректность бизнес-логики конструктора виджета.

Кроме того, при анализе сетевой активности не прослеживается явный и легко отслеживаемый бизнес-API-запрос, через который можно было бы проверить передачу параметров конфигурации виджета:
- `theme`
- `event_type`
- `event_lang`
- `width`
- `height`

### Почему это важно
- снижается прозрачность механизма генерации preview;
- усложняется диагностика и локализация дефектов;
- корректность применения настроек приходится проверять косвенно — через визуальный анализ preview и содержимое generated iframe-code;
- усложняется проведение QA-проверок.

### Предложение
- сделать процесс генерации preview более прозрачным для анализа;
- добавить явный preview URL или отдельный debug-friendly блок с финальными параметрами;
- упростить отслеживание параметров, участвующих в построении preview;
- предусмотреть более понятную связь между настройками интерфейса и итоговым generated code.

## 5. Улучшение поведения формы при перезагрузке страницы

### Проблема
Требует уточнения ожидаемое поведение формы при перезагрузке страницы: должны ли сохраняться ранее выбранные пользователем параметры.

### Почему это важно
- если пользователь случайно обновит страницу, он может потерять введённые настройки;
- отсутствие сохранения состояния может ухудшать пользовательский опыт;
- это особенно важно при большом количестве настроек.

### Предложение
Определить ожидаемое поведение и при необходимости реализовать сохранение состояния формы:
- в `localStorage`,
- в query parameters,
- или в session state.

---

## 6. Улучшение устойчивости к одновременной нагрузке

### Проблема
Страница генерации виджета потенциально может использоваться несколькими пользователями одновременно, однако поведение под конкурентной нагрузкой отдельно не проверялось. Требуется проводить нагрузочное тестирование.

### Почему это важно
- возможны задержки генерации preview;
- может ухудшаться отклик интерфейса;
- могут возникать проблемы с зависимыми ресурсами или виджетом preview.

### Предложение
Провести нагрузочное тестирование и оценить:
- стабильность генерации preview;
- время отклика интерфейса;
- корректность формирования generated code при одновременном использовании формы несколькими пользователями.
