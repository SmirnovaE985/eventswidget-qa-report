# QA Report

## 1. Основная информация
- URL: https://dev.3s.info/eventswidget/
- Testing type: Исследовательское тестирование
- Tester: Елена Смирнова
- Date: 17.06.2026

## 2. Цель
Провести исследовательское тестирование страницы конструктора календаря мероприятий, проверить основные пользовательские сценарии, генерацию preview и embed-кода, корректность передачи параметров, включая обязательную проверку темы «Красная».

## 3. Окружение
- OS: Windows
- Browser: Chrome / Edge
- Viewport: Desktop
- Tools: Browser DevTools, Network, Console

## 4. Тестовые сценарии
В ходе тестирования были проверены следующие сценарии:
- загрузка страницы;
- выбор тематики;
- выбор страны;
- настройка ширины и высоты блока, анализ граничных значений;
- выбор цветовой темы;
- генерация preview;
- анализ generated iframe-code;
- проверка параметров `theme`, `event_type`, `event_lang`;
- поведение при выборе `width=100%` и `height=100%`;
- оценка UX/UI и адаптивности.

## 5. Особая проверка: Red Theme
Согласно заданию, был выполнен сценарий выбора цветовой темы «Красная» с последующей генерацией preview и анализом generated code.  
Цель проверки — убедиться, что в generated code параметр `theme` принимает корректное значение, соответствующее выбранной теме.

## 6. Основные выводы
По результатам тестирования установлено:
- выбранная цветовая тема передаётся в generated code через параметр `theme`;
- выбранные тематики передаются через параметр `event_type`;
- generated code в целом отражает выбранные параметры конфигурации;
- при выборе режимов `width=100%` и `height=100%` preview визуально не всегда отражает ожидаемое адаптивное поведение;
- анализ Network показал отсутствие явного бизнес-API-запроса, через который можно было бы удобно отслеживать передачу параметров виджета.

## 7. Edge Cases
Дополнительно были рассмотрены граничные и нетипичные сценарии:
- множественный выбор тематик;
- переключение между несколькими цветовыми темами;
- проверка относительных размеров (`100%`);
- анализ соответствия между UI, preview и generated code;
- оценка поведения страницы в условиях аналитического шума в Network.

## 8. Адаптив 
При выборе опций “на всю ширину контейнера” и “на всю высоту блока” generated code формируется с `width="100%"` и `height="100%"`, однако preview не демонстрирует ожидаемое растяжение на весь доступный контейнер. Это создаёт расхождение между настройками конструктора и визуальным предпросмотром.

## 9. Console / Network 
В DevTools Network были обнаружены преимущественно аналитические запросы (`mc.yandex.ru`), связанные с Яндекс.Метрикой и вебвизором. При этом не был выявлен явный бизнес-API-запрос, через который можно было бы однозначно отследить передачу параметров `theme`, `event_type`, `event_lang` при генерации preview. Это усложняет QA-анализ и диагностику возможных дефектов.

## 10. Conclusion
Страница позволяет настроить и сгенерировать код виджета, однако были выявлены особенности и потенциальные проблемы, связанные с прозрачностью генерации preview, интерпретацией параметров и поведением адаптивных режимов в preview. Рекомендуется улучшить наблюдаемость механизма генерации и сделать поведение относительных размеров более предсказуемым для пользователя.
